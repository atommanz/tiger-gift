# Working Backwards Document
 
## โปรเจกต์: FTC Gift Finder — ระบบแนะนำของขวัญอัจฉริยะ
 
**Hackathon:** 1 วัน (4-5 ชั่วโมง) | **ทีม:** 3 คน
 
---
 
## 1. Who is the customer? (ลูกค้าคือใคร?)
 
**Primary Customer:** คนวัยทำงาน อายุ 20-45 ปี ที่ต้องการซื้อของขวัญให้คนรู้จักแต่คิดไม่ออกว่าจะให้อะไรดี
 
**ลักษณะของลูกค้า:**
 
- คนที่กำลังจะไปงานวันเกิด งานปีใหม่ งานเลี้ยง หรือโอกาสพิเศษต่างๆ แต่ไม่รู้จะซื้ออะไร
- คนที่เบื่อกับการให้ของซ้ำๆ เช่น กระเช้าผลไม้ รังนก กล่องคุ้กกี้
- คนที่ไม่มีเวลาเดินห้าง หรือเดินแล้วก็ยังเลือกไม่ได้
- คนที่อยากให้ของขวัญที่มีความหมายและแตกต่างจากคนอื่น
 
**Secondary Customer:** ทีม Marketing ของ Index Living Mall / Flying Tiger Copenhagen (FTC) ที่ต้องการเพิ่มยอดขายผ่านช่องทางออนไลน์
 
---
 
## 2. What is the customer problem/opportunity? (ปัญหาหรือโอกาสของลูกค้าคืออะไร?)
 
**ปัญหาหลัก:**
 
- **คิดไม่ออก:** ทุกครั้งที่มีโอกาสพิเศษ (ปีใหม่ วันเกิด วันครบรอบ) ลูกค้าใช้เวลานานในการตัดสินใจว่าจะซื้อของขวัญอะไรดี
- **ของซ้ำซาก:** สุดท้ายก็วนลูปกลับไปซื้อกระเช้าผลไม้ รังนก หรือจับของที่ไม่ได้ตั้งใจจริงๆ อย่างกล่องคุ้กกี้
- **เสียเวลา:** ต้องไปเดินห้างทั้งวัน แต่สุดท้ายก็ยังเลือกไม่ได้
- **ไม่รู้จัก FTC:** ทั้งที่ Flying Tiger Copenhagen มีสินค้า gift ที่น่ารัก มีดีไซน์ หลากหลาย และราคาไม่แพง แต่ลูกค้าจำนวนมากยังไม่รู้จักหรือไม่เคยเข้าร้าน
 
**โอกาส:**
 
- สร้างช่องทางการค้นพบสินค้า FTC ผ่าน experience ที่สนุกและง่าย
- เปลี่ยน "ความลังเล" ของลูกค้าให้เป็น "การตัดสินใจ" ภายในไม่กี่นาที
- เพิ่ม conversion จากคนที่มี intent ซื้อของขวัญแต่ไม่รู้จะซื้ออะไร
 
---
 
## 3. What is the solution? (วิธีแก้ปัญหาคืออะไร?)
 
**FTC Gift Finder** — เว็บแอปพลิเคชันที่ช่วยแนะนำของขวัญจาก Flying Tiger Copenhagen โดยใช้ข้อมูลเบื้องต้นจากผู้ใช้
 
**หลักการทำงาน:**
 
1. ผู้ใช้ตอบคำถามสั้นๆ ทีละ step (เพศผู้รับ, ช่วงอายุ, ความสัมพันธ์, โอกาส, งบประมาณ, สไตล์)
2. ระบบประมวลผลและแนะนำสินค้า 15-20 ชิ้นที่เหมาะสม
3. ผู้ใช้เลื่อนดูสินค้าแบบ TikTok-style (swipe ขึ้น-ลง)
4. เลือกสินค้าใส่ตะกร้า → เขียนการ์ดอวยพร → Checkout → จบด้วยพลุ celebration
 
**จุดเด่น:**
 
- UX ที่สนุก ง่าย และรวดเร็ว — จากไม่รู้จะซื้ออะไร สู่การ checkout ภายใน 3-5 นาที
- แนะนำสินค้าที่ตรง persona ของผู้รับ ไม่ใช่ random
- ประสบการณ์เหมือนมีเพื่อนช่วยเลือกของขวัญให้
 
---
 
## 4. What's the customer experience? (ประสบการณ์ลูกค้าเป็นอย่างไร?)
 
**User Journey:**
 
**Step 1 — เลือกข้อมูลผู้รับของขวัญ (Multi-step Form)**
 
- หน้าจอ 1: เพศของผู้รับ (ชาย / หญิง / ไม่ระบุ)
- หน้าจอ 2: ช่วงอายุ (เด็ก / วัยรุ่น / วัยทำงาน / ผู้ใหญ่)
- หน้าจอ 3: ความสัมพันธ์ (แฟน / เพื่อน / พ่อแม่ / เจ้านาย / เพื่อนร่วมงาน)
- หน้าจอ 4: โอกาส (วันเกิด / ปีใหม่ / ขึ้นบ้านใหม่ / แสดงความยินดี / ขอบคุณ)
- หน้าจอ 5: งบประมาณ (ต่ำกว่า 300 / 300-500 / 500-1,000 / 1,000+)
- หน้าจอ 6: สไตล์ (น่ารัก / มินิมอล / สนุกสนาน / practical)
- ทุก step มีปุ่ม "ถัดไป" พร้อม progress bar ด้านบน
 
**Step 2 — Loading & Reveal**
 
- หน้าจอ loading animation สวยๆ พร้อมข้อความ "กำลังเลือกของขวัญให้คุณ..."
- สร้างความตื่นเต้นและ anticipation
 
**Step 3 — สินค้าแนะนำ (TikTok-style Feed)**
 
- แสดงสินค้าทีละชิ้น เต็มจอ พร้อมรูปภาพ ชื่อ ราคา
- เลื่อนขึ้น-ลงเพื่อดูสินค้าถัดไป (snap scroll)
- ปุ่ม "Add to Cart" ชัดเจนในแต่ละหน้า
- ไอคอนตะกร้ามุมขวาบน แสดง badge จำนวนสินค้า (max 99)
 
**Step 4 — ตะกร้าสินค้า**
 
- รายการสินค้าที่เลือก พร้อมรูป ชื่อ ราคา
- ปุ่ม +/- ปรับจำนวน
- ยอดรวมทั้งหมด
- ช่องเขียนการ์ดคำอวยพร (text area)
- ปุ่ม "Checkout"
 
**Step 5 — Celebration**
 
- เมื่อกด Checkout แสดง confetti/พลุ animation เต็มจอ
- ข้อความ "ของขวัญพร้อมส่งแล้ว! 🎉" (จบ demo prototype)
 
---
 
## 5. How will you measure success? (วัดผลความสำเร็จอย่างไร?)
 
**สำหรับ Hackathon Demo (วันนี้):**
 
- ✅ Prototype ทำงานได้ครบ flow ตั้งแต่เลือกข้อมูล → ดูสินค้า → ใส่ตะกร้า → checkout → พลุ
- ✅ UX ลื่นไหล ไม่มีจุดสะดุด ใช้งานได้ใน 3 นาที
- ✅ ดูสวยงาม น่าใช้ สอดคล้องกับ brand identity ของ FTC
- ✅ กรรมการเข้าใจ concept ทันทีเมื่อเห็น demo
 
**สำหรับ Production (ถ้าพัฒนาต่อ):**
 
- Completion rate ของ flow (จาก step 1 จนถึง checkout) > 40%
- Average time to checkout < 5 นาที
- Add to cart rate > 30% ของคนที่เข้ามาดูสินค้า
- Customer satisfaction score > 4/5
- เพิ่ม traffic เข้าหน้า FTC product page บนเว็บ Index Living Mall
 
---
 
## 6. What needs to be built? (ต้องสร้างอะไรบ้าง?)
 
**Tech Stack แนะนำ (สำหรับ 4-5 ชม.):**
 
- Frontend: React / Next.js หรือ vanilla HTML+CSS+JS (แล้วแต่ความถนัดทีม)
- Animation: CSS animations / Framer Motion / Lottie
- Data: JSON file สำหรับ product catalog (mock data)
- Deploy: Vercel / Netlify (ถ้ามีเวลา)
 
**สิ่งที่ต้องสร้าง:**
 
1. **Multi-step form** — 6 หน้าจอ พร้อม progress bar และ transition animation
2. **Loading screen** — animation ระหว่างประมวลผล (2-3 วินาที)
3. **Product feed (TikTok-style)** — snap scroll แสดงสินค้าทีละชิ้น เต็มจอ
4. **Add to Cart** — ปุ่มเพิ่มสินค้า + badge counter บนไอคอนตะกร้า
5. **Cart page** — รายการสินค้า, ปรับจำนวน, ยอดรวม, ช่องเขียนการ์ด
6. **Checkout celebration** — confetti/fireworks animation
7. **Product data** — mock data สินค้า FTC 20-30 รายการ (รูป, ชื่อ, ราคา, tags)
 
**แบ่งงาน (แนะนำ 3 คน):**
 
- คนที่ 1: Multi-step form + Loading screen
- คนที่ 2: Product feed (TikTok scroll) + Add to Cart logic
- คนที่ 3: Cart page + Checkout celebration + เตรียม mock data & รูปสินค้า
 
---
 
## 7. What will you NOT build? (สิ่งที่จะไม่ทำ?)
 
- ❌ ระบบ login / สมัครสมาชิก
- ❌ ระบบชำระเงินจริง (payment gateway)
- ❌ Backend / Database จริง
- ❌ ระบบ AI/ML สำหรับ recommendation จริง (ใช้ filter logic ง่ายๆ แทน)
- ❌ Responsive ครบทุก device (focus mobile-first อย่างเดียว)
- ❌ ระบบจัดส่ง / ติดตามพัสดุ
- ❌ Admin panel สำหรับจัดการสินค้า
- ❌ Search / filter สินค้าด้วยตัวเอง
- ❌ Review / rating สินค้า
- ❌ Wishlist / Share link
 
---
 
## 8. Anything else? (อื่นๆ)
 
**Feature เพิ่มเติมที่แนะนำ (ถ้ามีเวลาเหลือ):**
 
- **"Surprise Me" button** — ปุ่มสุ่มของขวัญ 1 ชิ้นสำหรับคนที่ขี้เกียจเลือกเอง
- **Gift wrapping preview** — แสดง preview ว่าของขวัญห่อแล้วจะหน้าตาเป็นยังไง (mock image)
- **Share result** — ปุ่มแชร์ลิสต์ของขวัญที่เลือกไปให้เพื่อนดู (copy link)
- **Sound effect** — เสียงเล็กๆ ตอน add to cart และตอนพลุ เพิ่มความสนุก
 
**ความเสี่ยงและแผนรับมือ:**
 
| ความเสี่ยง | แผนรับมือ |
|---|---|
| เวลาไม่พอ | ตัด feature เสริมออก focus แค่ core flow ให้ smooth |
| หารูปสินค้าไม่ทัน | ใช้ placeholder image + ชื่อสินค้าจริง |
| TikTok scroll ทำยาก | fallback เป็น card grid แทน |
| Animation หนัก | ใช้ CSS transition ธรรมดาแทน library |
 
**Tips สำหรับ Demo:**
 
- เตรียม script การ demo ไว้ล่วงหน้า เล่า story ให้กรรมการเห็นภาพ
- เริ่ม demo จาก pain point → solution → live walkthrough
- ใช้ข้อมูลจริงของ FTC (ชื่อสินค้า ราคาจริง) เพื่อให้ดู realistic
- ถ้ามีเวลา ใส่ branding ของ FTC (สี ฟอนต์ โลโก้) เพื่อให้ดู production-ready
